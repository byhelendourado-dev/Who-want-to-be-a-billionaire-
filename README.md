# Who Wants to Be a Billionaire

Execute python main.py to run the game.

Explanation:
For my final project, I created a game inspired by Who Wants to Be a Millionaire. It's multiple-choice questions game that increase in difficulty as the player progress through the levels. The objective is to answer all questions correctly to win the game.
I used AI tools to help me with the structure of the project, but I did all the rest, coding and classes.
My dad helped me to learn many things. And also, to create the screens using tkinter, I used claude. However, I made sure to understand how the code worked before integrating it into my project. I modified and adapted the screen code so that it behaved the way I wanted and matched the functionality of my game.
All of the game logic, including managing questions, difficulty levels, checking answers, handling the game flow, working with JSON files, and implementing the object-oriented structure of the program, was coded by me. I used AI as a learning and support tool rather than relying on it to build the entire project.
This project helped me strengthen my understanding of object-oriented programming, program organization, debugging, and integrating different components into a complete application.
My initial proposal was to create a Pokémon game, but when I talked to my dad about it, he said it would be one of the most difficult options because of how complex it is. So I decided to switch to “Who Wants to Be a Millionaire?”, since it's easier to develop while still having a great final result.
During the programming process, one of the main challenges I faced was working with the Tkinter screens. Since I did not write the original interface code, I had to spend time understanding how it worked before I could make any changes. This included figuring out how the screens were connected, how the buttons functioned, and how the questions were displayed in the game.
After I understood the structure better, I was able to make changes to adapt the game to my ideas and fix parts that were not working properly. I also had to test the program several times to make sure the transitions between screens worked correctly and that the game did not show any errors while running.
There is one thing I'm not satisfied with, the help buttons. I didn't have time to work on them to make them work, so they are useless.
